Ключевые принципы:
- Симплекс-таблица хранит систему уравнений вида Ax = b, где базисные переменные выражены через небазисные.
- Строка цели: z – Σcⱼxⱼ = значение → в таблице: [–c₁, –c₂, ..., значение]
- Оптимальность (максимизация): все коэффициенты в строке цели ≥ 0
- Входящая переменная: наиболее отрицательный коэффициент (даёт наибольший рост z)
- Исходящая переменная: минимальное отношение RHS / pivot_col (сохраняет неотрицательность)

### Общий план решения (по шагам)
Шаг 1. Чтение данных из файла.

Формат:
- objective: c1 c2 ... cn — коэффициенты максимизируемой функции
- constraint: тип a1 a2 ... an b — ограничение вида
- a1·x1 + … + an·xn (≤ / == / ≥) b.

Все переменные по умолчанию: xᵢ ≥ 0.

Шаг 2. Преобразование к стандартной форме (с дополнительными переменными)
Для каждого ограничения:
≤ → добавить slack-переменную s ≥ 0: ... + s = b.

≥ → добавить избыточную sur ≥ 0 и искусственную a ≥ 0: ... – sur + a = b.

== → добавить искусственную a ≥ 0: ... + a = b.

Как результат получим все переменные ≥ 0, все ограничения - равенства.

Шаг 3. Построение вспомогательной задачи (Фаза I)
Цель: минимизировать сумму искусственных переменных ⇔ максимизировать W = –(a₁ + a₂ + …)
Начальная симплекс-таблица:

1) Строки - это преобразованные ограничения (с RHS в последнем столбце)
2) Последняя строка (W): коэффициенты = +1 при aᵢ, 0 иначе

 Далее необходимо вычесть все строки, где aᵢ в базисе, чтобы выразить W через небазисные переменные. Принцип основывается на начальном базисном решении - slack и artificial переменные. Нужно вывести все aᵢ из базиса и добиться W = 0.

Шаг 4. Решение Фазы I симплекс-методом
Цикл:

Если все коэффициенты в строке W ≥ 0 → оптимум достигнут.
Иначе: выбрать наиболее отрицательный коэффициент → входящая переменная.
Вычислить минимальное отношение RHS / коэффициент > 0 → исходящая переменная.
Выполнить pivot (Жорданово исключение).
Повторить.

После завершения:

Если W ≠ 0 → нет допустимого решения.
Если W = 0 → переходим к Фазе II.

Шаг 5. Подготовка к Фазе II
Необходимо удалить все столбцы искусственных переменных, а затем построить новую целевую строку для исходной функции:
z = c₁x₁ + … + cₙxₙ
Исключить базисные переменные из строки z (вычесть линейную комбинацию строк ограничений).
В итоге можно получить новую симплекс-таблицу без aᵢ.
Важным моментом является то, что длина базиса должна совпадать с числом ограничений. Если после удаления aᵢ базис неполный - восстановить по единичным столбцам.

Шаг 6. Решение Фазы II симплекс-методом
Тот же алгоритм, что и в Фазе I, но:

Целевая функция - исходная (z вместо W) и искусственных переменных нет.

Цикл:

Если все коэффициенты в строке z ≥ 0 → оптимальное решение найдено.
Иначе - выбрать входящую/исходящую переменную, выполнить pivot.
Повторить.

Шаг 7. Извлечение и вывод результата
Оптимальное значение: z* = tableau[-1][-1].

Значения переменных:
Если переменная в базисе → её значение = RHS соответствующей строки
Иначе → 0
Вывести только x₁, x₂, … (игнорировать s, sur)
